# 财务APP \- 财务沙盘模块 PRD（优化版·含结果存储与解析）

**版本**：V1\.0 最终版 **适用**：前端/后端/测试直接开发 **数据库**：SQLite（本地） **核心约束**：不复制真实数据、仅14种固定场景、只存推演记录与结果、支持二次查看

# 1\. 产品概述

## 1\.1 定位

财务沙盘是离线情景推演工具，基于用户当前真实资产/负债/收支数据做财务压力测试与规划模拟，不修改真实账本、不复制全量数据，核心价值是帮助用户预判财务风险、优化财务规划。

## 1\.2 核心规则（强制）

- 仅支持14种内置预设场景，不允许用户新增、修改场景参数或自定义场景

- 推演时实时读取用户当前真实账本数据（资产、负债、收支），不备份整套账本数据，避免占用本地存储

- 仅保存核心数据：推演基础记录\+用户输入参数\+推演结果数据\+图表渲染数据，不存储冗余信息

- 支持历史推演记录的列表查看、详情二次查看、软删除操作，不支持物理删除

- 全程本地计算、本地存储，不上云、不发起任何网络请求，保障用户财务数据安全

## 1\.3 14种固定场景清单

1\. 失业维持时长推演（场景1，scene\_type=1）
2\. 债务利率上涨推演（场景2，scene\_type=2）
3\. 股票/基金下跌推演（场景3，scene\_type=3）
4\. 每月多存流动资金推演（场景4，scene\_type=4）
5\. 提前还清负债推演（场景5，scene\_type=5）
6\. 收入上涨推演（场景6，scene\_type=6）
7\. 大额一次性支出推演（场景7，scene\_type=7）
8\. 停止所有投资（全部清仓）推演（场景8，scene\_type=8）
9\. 每月多还负债推演（场景9，scene\_type=9）
10\. 被动收入上涨推演（场景10，scene\_type=10）
11\. 固定支出下降推演（场景11，scene\_type=11）
12\. 新增固定支出推演（场景12，scene\_type=12）
13\. 提前退休推演（场景13，scene\_type=13）
14\. 债务利率下降推演（场景14，scene\_type=14）

# 2\. 数据库表结构（直接建表）

## 2\.1 沙盘推演历史表

**表名**：sandbox\_history（存储推演基础记录，用于历史列表展示）

```sql
CREATE TABLE sandbox_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
    scene_type INTEGER NOT NULL,                -- 场景编号 1~14，对应14种固定场景
    scene_name VARCHAR(64) NOT NULL,           -- 场景名称，与scene_type一一对应
    user_title VARCHAR(100),                   -- 用户自定义标题（可选，方便用户区分不同推演）
    simulate_time VARCHAR(20) NOT NULL,        -- 推演时间 yyyy-MM-dd HH:mm:ss
    params TEXT NOT NULL,                      -- 用户输入参数，JSON格式存储
    result_desc VARCHAR(255),                  -- 推演结果描述（用于历史列表展示，简洁概括核心结果）
    created_at VARCHAR(20) NOT NULL,           -- 记录创建时间（与推演时间一致）
    is_deleted INTEGER DEFAULT 0 NOT NULL,     -- 软删除标识：0=正常 1=已删除
    remark VARCHAR(255)                       -- 用户备注（可选，用户可记录推演目的、备注等）
);
```

## 2\.2 沙盘推演结果表

**表名**：sandbox\_result（存储详细推演结果，与历史表关联，用于详情展示）

```sql
CREATE TABLE sandbox_result (
    id INTEGER PRIMARY KEY AUTOINCREMENT NOT NULL,
    history_id INTEGER NOT NULL,               -- 关联 sandbox_history.id，建立推演记录与结果的关联
    net_assets DECIMAL(16,2),                  -- 推演结束时的最终净资产（资产-负债）
    net_assets_change TEXT,                    -- 净资产变动说明，文字描述变动金额及原因
    cash_flow_monthly DECIMAL(16,2),           -- 推演后的月现金流（月收入-月支出-月负债还款）
    cash_flow_change TEXT,                     -- 现金流变动说明，文字描述变动金额及原因
    debt_pressure_level INTEGER,               -- 负债压力等级：1=健康 2=良好 3=注意 4=风险
    debt_pressure_desc TEXT,                   -- 负债压力详细描述，说明压力等级判定依据
    survival_months INTEGER,                   -- 可支撑月数（仅失业、提前退休场景使用，其余场景存NULL）
    interest_save DECIMAL(16,2),               -- 节省总利息（仅提前还款、利率下降等场景使用，其余存NULL）
    monthly_payment_change DECIMAL(16,2),      -- 月供变动额（正数=增加，负数=减少，仅负债相关场景使用）
    total_interest_change DECIMAL(16,2),       -- 总利息变动额（正数=增加，负数=减少，仅负债相关场景使用）
    chart_x TEXT NOT NULL,                     -- X轴数据（月份/时间节点），JSON数组格式存储
    chart_net_assets TEXT NOT NULL,            -- 净资产曲线数据，JSON数组格式存储，与chart_x一一对应
    chart_cash_flow TEXT NOT NULL,             -- 现金流曲线数据，JSON数组格式存储，与chart_x一一对应
    chart_debt TEXT NOT NULL,                  -- 负债曲线数据，JSON数组格式存储，与chart_x一一对应
    conclusion TEXT NOT NULL,                  -- 推演结论，简洁明了，前端直接展示给用户
    created_at VARCHAR(20) NOT NULL,           -- 结果创建时间（与推演时间一致）
    FOREIGN KEY (history_id) REFERENCES sandbox_history(id)  -- 外键关联，删除历史记录时联动删除结果
);
```

# 3\. 功能流程

1. 用户进入财务沙盘模块，系统展示14种固定场景，用户选择其中一种场景

2. 用户根据场景提示，输入自定义参数（如金额、百分比、选中负债/投资品等），确认参数无误后提交推演请求

3. 系统实时读取用户当前真实账本数据（资产、负债、收支等），结合用户输入参数，执行本地推演计算

4. 计算完成后，系统将推演基础记录（含结果描述）保存到sandbox\_history表，将详细推演结果保存到sandbox\_result表，两表通过history\_id关联

5. 前端读取sandbox\_result表中的数据，解析JSON格式的图表数据和文本结果，渲染推演结论、核心指标及折线图

6. 用户可在历史记录列表查看所有未删除的推演记录（展示scene\_name、simulate\_time、result\_desc等），点击某条记录可二次查看详情，也可执行软删除操作

# 4\. 14种场景开发明细（含结果存储\+解析展示）

## 场景1：失业维持时长推演（scene\_type=1）

### 4\.1\.1 用户输入（params存储示例）


- 当前净资产：从资产表和负债表计算得出（总资产合计 \- 总负债合计）

- 月被动收入：从资产表读取，含存款利息、投资分红、租金等无需主动工作获得的收入

- 月负债还款总额：从负债表读取，所有负债的月还款额合计（等额本息/等额本金计算后的月还款额）

## 场景3：股票/基金下跌推演（scene\_type=3）

### 4\.3\.1 用户输入（params存储示例）

```json
{
  "investment_ids": [1,3],  // 选中的股票/基金ID（多只可多选，从投资表中获取对应ID）
  "decline_rates": [20, 15],  // 对应投资品的跌幅（%），与investment_ids一一对应
  "hold_quantity": true  // 是否保持持仓（true=不卖出，继续持有；false=立即卖出，锁定亏损）
}
```
