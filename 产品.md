# 二、术语Glossary（新增）

|术语|定义|应用场景|
|---|---|---|
|FIFO|先进先出原则，即优先卖出/处理最早买入/创建的资产/记录|股票、基金卖出操作|
|等额本息|贷款还款方式，每月还款额固定，其中每月利息随剩余本金减少而减少，每月本金随剩余本金减少而增加|负债还款计算|
|等额本金|贷款还款方式，每月应还本金固定，每月利息随剩余本金减少而减少，因此每月还款额逐渐减少|负债还款计算|
|负债收入比|月度负债还款总额 ÷ 月度总收入 × 100%，用于评估用户负债压力|财务健康评估、额度评估|
|加权平均成本|多次买入资产后，按买入金额和数量计算的平均成本，公式为（累计买入总金额+累计手续费）÷累计持有数量|股票、基金成本计算|
|冲正|撤销已完成的操作，恢复操作前的资产、负债、账户余额状态，并生成对应冲正流水|所有流水操作的撤销|
|净资产|用户总资产减去总负债的差额，总资产包含账户余额、资产市值、通用资产（工资、租金等）|净资产增长率计算、财务健康评估|
# 三、核心架构

产品核心功能分为**基础账务层**、**分析决策层**、**财商教育层**、**隐私安全层**四大模块，所有功能均基于「本地存储、手动确认、无云端同步」核心原则实现。各模块联动逻辑清晰，确保账务闭环、数据准确。

# 四、功能规格详情（修正+补充版）

## 4.1 账户管理（Must）

### 4.1.1 功能定义

支持多类型账户的全生命周期管理，所有操作生成闭环流水，确保账务可追溯。

### 4.1.2 核心字段

|字段名|类型|约束|补充约束（新增）|备注|

|账户名称|字符串|非空、唯一|不区分大小写，支持中文、数字、下划线，禁止使用/、*、?等特殊字符，最大长度30字符|如“招商银行储蓄卡”“微信零钱”|
|账户类型|枚举|必选|无|现金/微信/支付宝/储蓄卡/社保卡等|
|账户余额|数值|≥0|单位：元，保留2位小数，四舍五入规则|单位：元，保留2位小数|
|备注|字符串|可选|最大长度200字符，支持换行|无|
### 4.1.3 核心操作（修正+补充）

|操作|触发条件|执行逻辑|补充逻辑（修正+新增）|
|---|---|---|---|
|新增账户|用户手动创建|初始化余额为0，生成“账户创建”流水记录|无|
|修改账户|用户编辑账户信息|仅修改名称/类型/备注，余额不可直接修改（需走“余额调整”流程）|修改账户名称时，需校验新名称唯一性，若重复提示“该账户名称已存在，请更换”|
|删除账户|账户余额为0|非空账户需先调整余额至0，删除后流水保留，账户标记为“已删除”（不可恢复）|虚拟数据账户（新手模板）可直接删除，无需调整余额至0，删除后虚拟流水同步删除|

|内部转账|用户手动发起，选择转出/转入账户、输入金额|扣减转出账户余额，增加转入账户余额，生成双向流水|补充失败场景处理：①余额不足时弹窗提示“转出账户余额不足，无法完成转账”，禁止转账；②转入账户已删除时，提示“目标账户不可用，请重新选择”；③转账过程中APP崩溃，重启后自动回滚金额，并生成“转账异常”流水，标注异常原因|
### 4.1.4 流水规则

- 所有账户操作（新增/修改/余额调整/转账/资产入账/股票/基金卖出）自动生成流水记录，无例外；

- 流水字段：操作时间、操作类型、账户、金额、变动后余额、备注、操作人（默认当前用户）、流水状态（正常/异常/冲正）；

- 流水不可删除，仅支持查看，冲正记录需关联原流水ID，标注“冲正原流水XXX”。

### 4.2.2 股票资产（Must）（修正+补充）

#### 核心字段

|字段名|类型|约束|补充约束|备注|
|---|---|---|---|---|
|股票名称|字符串|非空|最大长度20字符|-|
|股票代码|字符串|可选（预留字段）|支持数字、字母，最大长度10字符|便于未来扩展|
|交易类型|枚举|必选|无|买入/卖出|
|交易价格|数值|＞0|单位：元/股，保留3位小数，四舍五入|-|
|交易股数|数值|＞0，整数|无|-|
|手续费|数值|≥0|单位：元，保留2位小数，向上取整至分|单位：元，保留2位小数|
|交易时间|时间戳|非空|精确到分钟，不可晚于当前时间|精确到分钟|
#### 核心计算逻辑（修正+补充）

- 首次买入成本价：(买入价格×股数 + 手续费) ÷ 股数；

- 多次买入加权平均成本：(累计买入总金额 + 累计手续费) ÷ 累计持股数；

- 卖出逻辑（FIFO，补充部分卖出规则）：优先卖出最早买入的股票；部分卖出时，剩余股票保留原批次成本价，不重新计算加权成本（例：最早买入1000股，成本价10元/股，卖出500股后，剩余500股成本价仍为10元/股，后续买入批次单独记录成本）；

- 卖出金额=卖出价格×股数 - 手续费，卖出资金自动划入关联账户，生成“股票卖出入账”流水；

- 浮动盈亏：(当前实时价 - 成本价) × 剩余持股数；

- 市值：当前实时价 × 剩余持股数；

- 测试用例（新增）：首次买入100股，价格10元，手续费5元；二次买入200股，价格12元，手续费8元，验证加权成本=(100×10+5+200×12+8)/300=11.38元/股；卖出150股（优先卖出首次100股+二次50股），剩余150股成本价仍为12元/股。

#### 展示项

成本价、实时价（用户手动修改）、持股数、市值、浮动盈亏、历史交易记录（按时间倒序，区分买入/卖出批次）。

### 4.3.3 核心计算逻辑（修正+补充）

#### 等额本息还款公式（补充终止规则）

- 每月还款额 = 本金×月利率×(1+月利率)^期数 ÷ [(1+月利率)^期数 - 1]；

- 月利率 = 年化利率 ÷ 12；

- 每月利息 = 剩余本金×月利率；

- 每月本金 = 每月还款额 - 每月利息；

- 剩余本金 = 剩余本金 - 每月本金；

- 终止规则（新增）：当剩余本金≤0时，自动终止还款计划，标记负债为“已结清”，最后一期还款额调整为剩余本金+当期利息，避免多扣；

- 测试用例（新增）：本金100000元，年化利率3.5%，期数12期，月利率≈0.2917%，每月还款额≈8530.99元，最后一期剩余本金≈8506.67元，利息≈248.22元，还款额≈8754.89元，结清后剩余本金为0。

#### 等额本金还款公式（补充终止规则）

- 每月应还本金 = 本金 ÷ 期数；

- 每月利息 = 剩余本金×月利率；

- 每月还款额 = 每月应还本金 + 每月利息；

- 终止规则（新增）：同等额本息，剩余本金≤0时终止，标记为“已结清”；

- 测试用例（新增）：本金120000元，年化利率4.8%，期数12期，每月应还本金10000元，首月利息480元，还款额10480元，次月利息440元，还款额10440元，依次递减，最后一期利息40元，还款额10040元。

#### 特殊规则（补充）

- 助学贷款：支持“计息开关”，未毕业/免息期内不计息；免息期结束后自动开启计息，提示用户“免息期已结束，开始计息”；

- 亲友借款：支持“无凭证/无利息/无期限”，仅记录本金、剩余本金、还款记录；无期限时，不生成还款提醒，用户可手动发起还款；

- 提前还款：用户发起后，重算剩余本金和后续还款计划（等额本息/本金均支持）；补充超额还款规则：允许还款金额超过剩余本金，超额部分自动冲抵“未结利息”（如有），仍超额则生成“负债提前结清+超额退款”流水，将超额金额退回绑定账户；

- 跨月还款日适配（补充）：如还款日为30日，2月无30日则自动适配为2月最后一天，3月还款日仍为30日，不改变原还款日设置。

|操作|执行逻辑|补充逻辑（新增）|
|---|---|---|

## 4.4 资产-负债-账户联动（Must）（补充）

### 4.5.1 评估维度（补充净资产计算）

|---|---|---|

1. ### 4.8 净资产增长率仪表盘（Must）

   #### 4.8.1 核心公式（修正 + 补全）

   1. 净资产增长率 = (期末净资产 − 期初净资产) / 期初净资产 × 100%；
   2. 修正公式：净资产 = 总资产（账户余额 + 资产市值 + 通用资产） - 总负债（剩余本金）；
   3. 测试用例：期初净资产 100000 元（账户余额 50000 元 + 股票市值 30000 元 + 通用资产 30000 元 - 负债 10000 元），期末净资产 120000 元，增长率 =(120000-100000)/100000×100%=20%；
   4. 补充说明：期初 / 期末净资产计算需包含所有资产类型（账户、股票、基金、通用资产），负债仅计算剩余本金，不含已结清负债。

   #### 4.8.2 展示与更新规则（补全）

   1. 展示形式：仪表盘折线图展示，支持切换 “月度 / 季度 / 年度” 增长率，hover 可查看当期期初、期末净资产明细及计算过程；
   2. 更新频率：用户新增 / 修改资产、负债、账户余额后，自动实时更新增长率数据；
   3. 历史数据：保留近 3 年增长率记录，支持按时间筛选查看，可本地导出明细（导出格式支持 Excel/CSV，脱敏处理）。

   1. 目标创建：用户选择目标类型、录入金额、期限，系统自动计算每月需投入金额（公式：每月投入金额 = 目标金额 ÷ 目标期限，若有余数，最后一个月补足）；
   2. 进度跟踪：实时展示目标完成率（完成率 = 累计投入金额 ÷ 目标金额 ×100%），可视化进度条展示，同步显示剩余期限、剩余需投入金额；

   3. 目标金额修改：若已累计投入金额＞修改后目标金额，弹窗提示 “累计投入已超过新目标金额，是否直接标记为已完成？”。
