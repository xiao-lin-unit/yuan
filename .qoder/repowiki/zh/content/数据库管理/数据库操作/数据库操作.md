## 详细组件分析

### DatabaseManager 类详解
- 单例模式：全局仅维护一个数据库实例，避免重复连接
- 连接管理：支持原生与 Web 两套路径；Web 环境支持从 localStorage 加载持久化数据
- 查询与执行：query()、run()、batch() 统一使用位置参数（数组）绑定，避免字符串拼接引发 SQL 注入
- 事务：executeTransaction() 直接委托底层 executeSet；同时提供手动事务封装示例
- 缓存：查询结果缓存（Map），命中则直接返回，降低重复查询成本
- 持久化：Web 环境采用防抖节流保存至 localStorage，减少频繁写入
- **新增**：索引优化：包含月度快照表的复合索引，提升查询性能

#### query() 查询方法
- 功能：执行 SELECT 查询，返回对象数组
- 参数绑定：使用位置参数（数组），避免字符串拼接
- 结果转换：原生平台返回字段名与值映射；Web 平台使用 stmt.getAsObject()
- 缓存：可选缓存命中后直接返回，提升读性能

### 数据库连接管理策略
- 单例模式：全局仅维护一个 db 实例，避免重复连接
- 连接状态检查：getStatus() 返回 isNative、connected、initialized、connecting、cacheSize
- 原生平台：通过 SQLiteConnection.createConnection()/retrieveConnection()/open() 管理连接
- Web 平台：通过 SQL.js 初始化内存数据库，支持从 localStorage 加载持久化数据
- 关闭连接：close() 会确保 Web 环境保存后再关闭，原生平台调用 closeConnection()

### Capacitor SQLite 与 SQL.js 运行模式对比
- Capacitor SQLite（原生）
  - 特点：原生数据库，支持事务、索引、外键约束
  - 适用：Android/iOS 原生应用
- SQL.js（Web）
  - 特点：纯 JS 内存数据库，需持久化到 localStorage
  - 适用：Web 浏览器、Electron（Web 渲染进程）
- 模式切换：通过 Capacitor.isNativePlatform() 自动判断，无需手动切换

## 月度财务快照管理功能

### 功能概述
月度财务快照管理功能是 Finance App 的重要增强，提供了自动化的财务数据持久化和历史趋势分析能力。该功能通过以下方式实现：

- **自动化采集**：基于 Day.js 的月度快照任务，自动计算总资产、总负债、净资产等关键指标
- **数据持久化**：将计算结果存储到 asset_monthly_snapshots 表中，支持历史数据追踪
- **趋势分析**：提供历史快照查询接口，支持6个月和12个月的时间范围分析
- **实时展示**：财务仪表板实时展示资产增长趋势和财务健康状况

### 快照计算逻辑
快照任务会自动计算以下财务指标：

1. **资产计算**：
   - 流动资金：非信用卡账户余额
   - 社保资产：社保账户金额
   - 公积金资产：公积金账户金额
   - 股票市值：股票数量 × 当前价格
   - 基金净值：基金份额 × 当前净值
   - 亲友借出：借出给亲友的金额

2. **负债计算**：
   - 总负债：所有负债剩余本金 + 信用卡已用额度

3. **净资产计算**：
   - 净资产 = 总资产 - 总负债

4. **已实现盈亏**：
   - 股票已实现盈亏 + 基金已实现盈亏

### 快照数据查询接口

财务仪表板提供了完整的快照数据查询接口：

1. **保存快照**：`saveSnapshot(year, month, data)`
2. **加载快照**：`loadSnapshot(year, month)`
3. **获取历史数据**：`getSnapshotHistory(months)`

### 财务指标计算

基于快照数据，系统可以计算多种财务指标：

1. **资产月增长率**：`(本月资产 - 上月资产) / 上月资产 × 100%`
2. **债务收入比**：`月供 / 月收入 × 100%`
3. **资产负债率**：`总负债 / 资产总额 × 100%`
4. **综合评分**：基于多个财务指标的综合评估

### 性能优化措施

1. **索引优化**：为 `asset_monthly_snapshots` 表的 `(year, month)` 字段创建复合索引
2. **查询缓存**：快照查询结果缓存，避免重复计算
3. **批量操作**：快照任务使用批量查询减少数据库往返
4. **时间范围限制**：默认只查询最近12个月的数据

## 性能考量
- 查询缓存：query() 支持 useCache，命中后直接返回，显著降低重复查询成本
- 防抖持久化：Web 环境通过防抖节流保存到 localStorage，减少频繁写入
- 批量执行：batch() 一次性提交多条语句，减少往返次数
- 索引优化：初始化时为高频查询字段建立索引，包括快照表的复合索引
- 事务批处理：将相关写操作放入事务，减少锁竞争与提交次数
- **新增**：快照数据缓存：财务仪表板对快照查询结果进行缓存
- **新增**：时间范围限制：快照查询默认限制12个月范围，避免大数据量查询

## 故障排查指南
- 连接失败
  - 检查 Capacitor 平台识别是否正确
  - 查看原生平台连接一致性检查与连接状态
- 查询异常
  - 确认 SQL 语法与参数绑定顺序一致
  - 检查缓存是否命中导致旧数据
- 执行异常
  - 确认参数为数组且顺序正确
  - 检查 Web 环境是否触发了防抖保存
- 事务异常
  - 手动事务注意在异常时执行 ROLLBACK
  - 使用 executeTransaction() 简化事务流程
- 数据持久化
  - Web 环境检查 localStorage 中是否存在 sqlite_db 名称的数据
- **新增**：快照任务异常
  - 检查 Day.js 日期计算是否正确
  - 确认快照表结构是否正确初始化
  - 验证财务指标计算逻辑
- **新增**：财务仪表板数据异常
  - 检查快照历史查询的时间范围设置
  - 确认图表数据格式转换是否正确

## 结论
DatabaseManager 通过统一的接口屏蔽了 Capacitor SQLite 与 SQL.js 的差异，提供了：
- 安全的参数绑定与 SQL 注入防护
- 事务与批量执行能力
- 查询缓存与持久化优化
- 明确的状态检查与异常处理
- **新增**：月度财务快照管理功能，支持自动化的财务数据持久化和历史趋势分析

在 Finance App 中，该设计使得业务层可以专注于领域逻辑，而不必关心底层数据库实现细节。新增的快照管理功能进一步增强了应用的财务分析能力，为用户提供全面的财务健康状况监控和趋势预测。
